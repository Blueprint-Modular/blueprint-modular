Troubleshooting

Dépannage courant pour les apps Blueprint Modular.

L'app ne démarre pas

  • Vérifier que Python 3.9+ est utilisé : python --version.
  • Vérifier l'installation : pip show blueprint-modular.
  • Vérifier que le fichier passé à bpm run existe et est exécutable (pas d'erreur d'import au démarrage).

Port déjà utilisé

Changer le port : bpm run app.py --server.port=8502 (ou l'option équivalente selon la CLI BPM).

Composants qui ne s'affichent pas

  • Vérifier la console du navigateur (erreurs JS, blocage d'iframe si sandbox).
  • S'assurer que l'URL du sandbox (sandbox.beam-consulting.fr) est accessible depuis le réseau du déploiement.

Session state perdu

Le session state est lié à la session utilisateur. En cas de rechargement forcé ou de déploiement sans persistance, l'état peut être réinitialisé. Pour des données persistantes, utiliser une DB ou un stockage externe.
